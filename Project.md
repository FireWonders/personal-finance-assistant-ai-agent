# 프로젝트의 진행 계획을 저장하는 파일
1. 목적 : 현재 내 재무 상태를 파악하고, 이를 통해 향후 1년간의 자금 운용에 대한 수입과 지출에 대한 계획을 수립함
2. 구현 방향 : react를 사용하여 frontend를 구축하고, 별도의 서버 업로드없이 excel, 혹은 Notion DB를 연동하여 로컬 또는 private network를 통해 구축함
 - 우선 접근 방식 : excel 혹은 csv방식으로 데이터를 저장하여 Notion DB에 업로드 가능한 방식으로 작성
 - 차선 : 우선 접근방식이 힘든 경우, 해당 폴더 내 로컬 파일을 생성하여 1차적으로 구현, 이후 외부로 export 가능한 형식으로 기능 구현
3. 웹 기반의 가벼운 프로젝트여야하며, 웹에서 내용 수정 및 편집, 지출 내역등의 업로드가 간편하여야 함.
4. 수입/지출내역은 금융결제원 오픈 api를 활용하여 작성하여야하며, api key의 경우 .env 파일을 통하여 관리하여야 함.
5. 개인정보 유출을 조심하기 위해 개인정보 파일은 한곳에 모아 .gitignore에 list 추가가능하여야 함.
6. 그 외의 구현 방식은 자율에 맡기되, 작성한 내용과 의도는 정리하여 본 Project 폴더 내 dev_log.md 파일에 기록하여야 함.
